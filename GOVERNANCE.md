# Governance

Shared Lists Starter is currently maintained by one primary maintainer. The project is pre-1.0, so the priority is keeping the app small, understandable, and safe to deploy.

## Decision Making

The maintainer decides what lands in the main repository. Pull requests are welcome when they improve the starter for other deployers and do not add private deployment assumptions.

Good changes usually:

- Improve security, privacy, accessibility, or deployment clarity.
- Keep OpenAI Sites and Cloudflare lanes separated by configuration.
- Add tests for auth, permissions, migrations, or user-visible behavior.
- Avoid turning the starter into a hosted multi-tenant SaaS.

## Contribution Policy

Contributions are accepted under the Apache License 2.0. By opening a pull request, you agree that your contribution is licensed under Apache-2.0.

This project does not currently require a CLA or DCO sign-off. If that changes, this file and `CONTRIBUTING.md` will be updated before the requirement is enforced.

## Releases

Releases use semver-style tags. Pre-1.0 releases may make breaking changes, but release notes should explain any migration work clearly.

## Maintainer Conduct

The maintainer should apply the same code of conduct to themselves that contributors are asked to follow.
